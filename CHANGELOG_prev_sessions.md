# SalesFlow — Changelog

## Сессия 19.06.2026 — построение B2B-логики

### Цель сессии
Автоматизировать сборку отчётов о продажах ключей по трём площадкам (B2B,
Chinaplay, GamersBase) из биллингового Universal Report в форматы листов
эталона `Продажи Б2Б`, `продажи_Чайна`, `продажи_GB ` (с пробелом).

### Состояние на конец сессии

**Достигнуто:**
- Chinaplay, GamersBase — работают, расхождения с эталоном ~1-2% за счёт
  усреднения курсов и корректировок
- B2B — собирается, **21 из 25** топ-партнёров эталона точно совпадают,
  **ID-маппинг 92%** через приоритет биллингового ID
- **FZE: 96/85 строк** (перепокрытие из-за разделения AlfaBank/GamersHub/MobiMoney
  в моём выходе — эталон сливает их в одну строку без указания платформы)
- **Wei** теперь точно ✓ через GE_KEEP_NAME
- Финальный файл: `Продажи_свод_04.2026_v12.xlsx`
- Streamlit-приложение: `salesflow_app.zip`

**Реализованные правила в engine.py:**

```python
ROKKY_KEEP_NAME = {"(EU) CiDiKi"}    # ROKKY-партнёры под своим именем
GE_KEEP_NAME    = {"Wei"}             # GE-партнёры под именем платформы
IGM_FZE_RIGHTS_HOLDERS = {"KRM"}      # (CIS) IGM + KRM → FZE
EXTRA_PARTNER_CATALOG = [
    ("FZE",   "AlfaBank",          "AlfaBank"),
    ("FZE",   "GamersHub",         "GamersHub"),
    ("FZE",   "MobiMoney",         "MobiMoney"),
    ("ROKKY", "(CIS) game-online", "(CIS) game-online"),
]
```

И исправление ID-маппинга: при наличии биллингового ID в каталоге — оставляем
его (приоритет «✓ по ID» над exact match по имени).

**Остаётся неавтоматизируемым:**
- **PRAGMATA** (219 ключей FZE) — в биллинге нет продаж, только закупка.
  Это продажа стока, идёт мимо Universal Report.
- **Точная группировка эталона** для FZE: моё разделение AlfaBank/GamersHub/MobiMoney
  даёт 96 строк, эталон сливает в 85 — разница на уровне отображения.
- **20-30 TM-строк** связаны с курсовой конверсией RUB→CNY и разделением партий.
- **53 ROKKY-строки** с (CIS) IGM/Metis Digital в Партнер2: в биллинге этих
  имён нет в Имя_платформы — эталон содержит данные не из Universal Report.

### Главные открытия

1. **Лист «Партнеры ROKKY »** (с пробелом) в эталоне — авторитетный список
   ROKKY-партнёров: (EU) CiDiKi, (EU) Difmark, (EU) Distribase SLU,
   (CIS) IGM, (EU) Metis Digital, Keysforgamers.com, Konrad_Hoscilo,
   GalaxyLink, Keybin, Difmark.com.
   Теперь подтягивается автоматически в engine.

2. **«ЮЛ» в каталоге партнёров** — это **наше** юр.лицо, с которым партнёр
   имеет договор:
   - ЮЛ=TM → Торнадо Менеджмент
   - ЮЛ=ROKKY → Rokky Limited
   - ЮЛ=GE → используется Название_1С (длинное юр.имя)
   - ЮЛ=FZE → нет в каталоге, отдельный канал

3. **Партнёры биллинга имеют префиксы**, отсутствующие в каталоге:
   `(CIS) (VAT) OZON` в биллинге vs `(VAT) OZON` в каталоге. Решено через
   normalize: убираем `(CIS)`, `(CN)`, `(EU)`, `(TRY)`, `(RU)`, `(TR)`,
   `(CNT)` и доменные суффиксы `.com`/`.ru`/`.io`/`.net`/`.org`/`.shop`.
   Сохраняем `(VAT)` — сильный признак российских партнёров (TM).

4. **Двойной маппинг для дублей в каталоге** (`(CIS) IGM` = TM и ROKKY):
   - Поиск по Имя_платформы → приоритет ROKKY
   - Поиск по Партнёр биллинга → приоритет TM

5. **Эмпирическое исключение** `ROKKY_KEEP_NAME`: `(EU) CiDiKi` в каталоге
   ROKKY, но в эталоне остаётся под своим именем (5 строк под именем CiDiKi,
   не ROKKY).

### Список открытых вопросов на следующую сессию

1. **ID-маппинг Bamboo → KRM для FZE-партнёров**: для PSN-карт в каталоге
   два дубля (19190557/Bamboo и 19191913/KRM). Биллинг даёт первый, эталон
   FZE — второй. Нужно правило: «для FZE-партнёров приоритет KRM-варианта».

2. **Источник стоковых продаж B2B** (PRAGMATA, частично PSN-карты) — где
   брать продажи, которых нет в Universal Report? Возможно отдельный отчёт
   из QuickBooks или 1С.

3. **Курсовая конверсия RUB→USD для FZE** — биллинг даёт RUB, эталон USD
   по тому же qty (для AlfaBank/GamersHub). Нужен курс или цена в USD
   из другого источника.

4. **«Игромир» = (CIS) IGM** — подтверждено пользователем что (CIS) IGM
   также может быть FZE. Однако в апрельском биллинге (CIS) IGM появляется
   только как Партнёр (не Имя_платформы). Возможно правило: 220 ключей
   (CIS) IGM с правообладателем KRM → FZE.

5. **Когда GE-партнёр оставляется под именем платформы**: Wei (PLATFORM, GE)
   остаётся 'Wei', а Giftery.pro (PLATFORM, GE) идёт под Prepaid Solutions
   FZCO. Правило неоднозначно.

6. **TM-конверсия RUB→CNY**: в эталоне 178 TM-строк все в CNY, в биллинге
   все в RUB/USD. Нужен курс конверсии или другой источник.

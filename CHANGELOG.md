## Сессия 25.06.2026 (доп. 7) — «События» НЕ выводятся из биллинга (внешний вход)

### Проверено: событие ≠ кросс-зонное перемещение биллинга
Гипотеза «событие = закуп ID в другой зоне» опровергнута: 7/25 положительных
событий Eneba объясняются закупом вне зоны; остальные 18 (Ship Graveyard 2,
Wall World, World War Z KR, Black Myth Wukong Deluxe…) имеют НУЛЕВОЙ закуп во
ВСЕХ 9 зонах биллинга, но положительное событие — ключи из источника, которого
в биллинге нет. Значит «События» — внутренние/ручные корректировки стока
(пулы, аллокации издателя, правки) из системы учёта ключей; из биллинга+выгрузок
НЕ выводятся (как и региональный сплит — факт склада).

### Следствие для архитектуры
«События» — ВНЕШНИЙ ВХОД (лог перемещений/корректировок), не движок. app.py уже
принимает его опциональной загрузкой [ID, События]. Для валидации — из эталона,
в production — из лога системы учёта. Движок строить не нужно.

Итог по входам авто-эталона:
- Загружено — вычисляется (KeyFlow) ✓
- Деньги/спрос — вычисляются (билдеры) ✓
- Региональный сплит — allocator + ревью (потолок ~98,6%)
- Остаток_нач — перенос (carry-over, к реализации)
- События — внешний лог (не вычислимо)
- Маппинг листинг→ID — персистентный (+ bundle-резолвер для Plati)

### Осталось
Перенос Остаток_конец → Остаток_нач между месяцами (единственный оставшийся
строящийся компонент).

## Сессия 25.06.2026 (доп. 10) — фикс двойного счёта bundle → Plati+GGSel 89.7%→94%

### Баг: двойной счёт спроса
suggest_from_raw определял несопоставленные листинги по СЫРОМУ `название товара`
(с зонной декорацией «РФ+СНГ | … | STEAM КЛЮЧ»), а билдер матчит по ЧИСТОЙ
колонке `Name`. Игры, уже сопоставленные билдером (HoMM Olden Era 4922),
резолвер поднимал заново (4932) → спрос удваивался, движение искажалось
(HoMM USD Продано 1001 vs эталон 412, Δ−589).

### Фикс + сопутствующее
- suggest_from_raw использует колонку `Name` (как билдер) → нет двойного счёта;
- размытые промо-бандлы делятся по наибольшему разрыву (а не всё в standard);
- EXCLUDE_KEYS сужен до вариативных пополнений (фикс-номинал карт — SKU каталога).
РЕЗУЛЬТАТ Plati+GGSel: конец 1948→2049/2172 (89.7%→94.3%), Σ|Δ| 5570→3276 (−41%).
bundle_demand очистился: 107 base / Σ 5599 (без задвоенных mapped-игр).
Остаток — фундаментальный регион×издание хвост (тот же ~98.6% потолок).

## Сессия 25.06.2026 (доп. 15) — построчная сверка QB-листа: лумп верен (выручка точна)

### Диагностика (снятие противоречия 99.6% движение vs 88% лист)
- Эталон: лист продаж == движение по ID ТОЧНО (Kinguin 1894/1894) — одна база.
- KeyFlow Загружено vs эталон: 100% по ID (1893/1893).
- Allocated продажи vs эталон: 98.5% по всем ID; ~87–94% на активно-проданных
  мультирегиональных (региональный потолок). «88%» ранее — доля по активным ID.

### split_sales_by_alloc → DEPRECATED
Проверка: разбитый лист теряет 33% ВЫРУЧКИ (Kinguin 83905.68→55952.59), т.к.
allocator разносит спрос лишь до наличия стока (остаток выпадает). Для QB
выручка обязана сохраняться точно → QB-лист продаж остаётся ЛУМПОМ (итоги и
одно-региональные ID точны). Региональное измерение для COGS — из ledger
движения, не из листа продаж. Функция помечена deprecated, в пайплайн не
подключена (app уже использует лумп — корректно).

### Итог

(для COGS) — в «Движении ключей» с потолком ~98.6%/ревью. Код продаж не менялся.
